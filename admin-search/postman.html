Using Cloudinary Postman Collections

For this example, we upload an image using the Node.js SDK. We include an array of tags that we can use as parameters for the Search API.

Click on the Postman link in the documentation.

In order to make you own API calls in Postman, you need to create an account and sign in. We see the Collections for all the APIs in Programmable Media.

Click on Search API to see all the endpoints. Then click on the fork icon located top right.

Create a Fork Label and a Workspace. I’ve named them "My Search API" and "My Workspace". Click on Fork Collection.

The API collection is set up to read Global variables in postman. Create an Environment with these global variables.

Click on New Environment.

Go to the Cloudinary dashboard and locate api_key, api_secret, and cloud_name. Enter these into the New Environment form. Note at the top left, these are global env variables. Mark the api_secret as “secret”, but cloud_name and api_key don’t need to be made secret.

Click on the “eye” next to the Environment. We’ll see an editable form with our environment variables. Our env set is still called “New Environment”.

On the left, click on Environments. Rename the "New Environment".

This environment has been renamed to "Cloudinary Training" to match the cloud name. The check mark indicates it is activated. Check it if it isn't.

Hover over the {cloud_name} variable in the API and it shows ‘cloudinary-training’ which it picked up from the global environment variable.

We're ready to add an expression for the search. Click on raw and we'll enter a JSON object.

Add a JSON object with a search expression. We’re looking for images with a ‘cat’ in the tag, uploaded within the last day and greater the 1MB in size.

The response is a JSON object. I shows that I have one asset with the tag 'cat' uploaded within the last day, that is greater than 1MB in size.
